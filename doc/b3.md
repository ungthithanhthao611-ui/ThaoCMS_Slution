2. TRUY VẤN DỮ LIỆU VỚI LINQ (CẦM TAY CHỈ VIỆC)
2.1. Lọc dành mục theo tham số (where) và sắp xếp (orderby)
Tình huống: Thay vì chỉ lấy bài viết của một chuyên mục cố định, chúng ta sẽ viết một hàm nhận vào một id bất kỳ. Khi người dùng truyền id nào, hệ thống sẽ lọc ra bài viết của chuyên mục đó và sắp xếp theo ngày đăng mới nhất.
A. Mã nguồn trong Controller (PostController.cs)
Sửa hàm Index (hoặc tạo hàm mới tên là ListByCategory) như sau:
// Tham số 'id' được truyền vào từ URL (ví dụ: /Post/Index/5)
public IActionResult Index(int? id) 
{
    // 1. Kiểm tra nếu không có id truyền vào thì trả về lỗi hoặc toàn bộ bài viết
    if (id == null) 
    {
        return BadRequest("Vui lòng cung cấp mã danh mục.");
    }

    // 2. Sử dụng LINQ với tham số 'id' linh hoạt
    var posts = _context.Posts
                .Where(p => p.CategoryId == id) 
                .OrderByDescending(p => p.CreatedDate) 
                .Include(p => p.Category) 
                .ToList();


    // 3. Truyền dữ liệu ra View
    return View(posts);
}

B. Giải thích chi tiết 
●	Tham số int? id: Dấu chấm hỏi ? cho phép id có thể bị rỗng (null) nếu người dùng không gõ số ID trên URL. Điều này giúp chương trình không bị lỗi "văng" ngay lập tức.
●	.Where(p => p.CategoryId == id): Đây là biểu thức Lambda. Hệ thống sẽ quét bảng Posts và chỉ chọn những dòng nào có cột CategoryId khớp chính xác với con số mà  truyền vào từ trình duyệt.
●	.OrderByDescending: Giúp bài viết vừa đăng xong sẽ hiện lên vị trí đầu tiên, giống như các trang báo điện tử hiện nay.
Tổng quát: công thức "vàng" này để áp dụng cho mọi bảng dữ liệu:
Biến_Kết_Quả = _context.Tên_Bảng
.Hàm_Lọc
.Hàm_Sắp_Xếp
.Hàm_Liên_Kết.Thực_Thi();
Giải thích chi tiết các "Mắt xích"
01.	_context.Posts: Điểm bắt đầu. Nó đại diện cho bảng dữ liệu trong SQL Server mà đã cấu hình ở Buổi 2.
02.	.Where(p => ...): Màng lọc dữ liệu.
a.	p đại diện cho một dòng trong bảng (Post).
b.	p.CategoryId == id là điều kiện lọc: "Chỉ lấy những bài viết có mã danh mục bằng với tham số truyền vào".
03.	.OrderByDescending(p => ...): Bộ sắp xếp.
a.	Dùng để đưa dữ liệu mới nhất (ngày lớn nhất) lên đầu trang.
04.	.Include(p => p.Category): Lệnh nạp chồng (Join bảng).
a.	Trong tài liệu anh soạn, bảng Post có liên kết với Category.
b.	Lệnh này bắt buộc phải có để lấy được Tên danh mục, nếu không thuộc tính này sẽ bị null và gây lỗi trang web.
05.	.ToList(): Lệnh chốt hạ.
a.	Nếu không có lệnh này, C# vẫn chưa thực sự tải dữ liệu về. Nó chỉ chính thức truy vấn SQL khi gặp lệnh .ToList().

3. THAO TÁC DỮ LIỆU CHUYÊN SÂU (CRUD)
3.1. Tìm kiếm và hiển thị chi tiết (Details)
Cập nhật Action Details trong PostController.cs:

// GET: Post/Details/5
public IActionResult Details(int id)
{
    // 1. Truy vấn bài viết theo ID
    // Sử dụng .Include(p => p.Category) để lấy kèm thông tin Danh mục (Join bảng)
    var post = _context.Posts
        .Include(p => p.Category)
        .FirstOrDefault(p => p.Id == id);

    // 2. Kiểm tra nếu không tìm thấy bài viết (tránh lỗi màn hình trắng)
    if (post == null)
    {
        return NotFound(); // Trả về trang lỗi 404
    }

    // 3. Truyền dữ liệu sang View
    return View(post);
}


3.2. Thêm mới dữ liệu (Create)
Mục tiêu của phần này là giúp nắm vững quy trình gửi dữ liệu từ trình duyệt về máy chủ và ghi dữ liệu đó vào Database. Chúng ta tạm thời chưa quan tâm đến việc người dùng nhập đúng hay sai (phần này sẽ học ở Buổi 5).
A. Code xử lý tại Controller (CategoryController.cs)
Trong Controller, chúng ta cần 2 hàm (Action). Một hàm để mở trang nhập liệu và một hàm để thực hiện việc lưu.
C#
// 1. Hàm GET: Dùng để hiển thị giao diện Form cho  nhập
[HttpGet]
public IActionResult Create()
{
    return View();
}

// 2. Hàm POST: Dùng để đón dữ liệu từ Form gửi lên và lưu vào SQL
[HttpPost]
public IActionResult Create(Category model)
{
    // BƯỚC 1: Thêm dữ liệu vào bộ nhớ tạm của Entity Framework
    _context.Categories.Add(model);

    // BƯỚC 2: Ra lệnh cho hệ thống ghi dữ liệu thật sự vào SQL Server
    _context.SaveChanges();

    // Sau khi lưu thành công, tự động quay về trang danh sách
    return RedirectToAction("Index");
}

B. Giải thích quy trình 2 bước (Bắt buộc phải nhớ)
hãy nhấn mạnh  rằng thiếu 1 trong 2 bước này, dữ liệu sẽ không bao giờ xuất hiện trong Database:
1.	Lệnh _context.Categories.Add(model): Đây là bước "Đăng ký". Dữ liệu từ các ô nhập liệu được gom lại thành một đối tượng Category và đưa vào danh sách chờ của hệ thống.
2.	Lệnh _context.SaveChanges(): Đây là bước "Chốt đơn". Chỉ khi lệnh này chạy, Entity Framework mới tạo ra câu lệnh SQL INSERT INTO... để đẩy dữ liệu vào bảng Categories trong SQL Server.

Tiếp theo ta cần code để thực hiện xóa 1 danh mục trong database
Trong ASP.NET Core MVC, quy trình xóa diễn ra qua 3 bước: Tìm đối tượng -> Xóa khỏi bộ nhớ tạm -> Lưu thay đổi vào CSDL.
A. Mã nguồn xử lý tại CategoryController.cs
Hướng dẫn thêm Action sau vào Controller:
C#
// Action nhận vào Id của danh mục cần xóa
public IActionResult Delete(int id)
{
    // Bước 1: Tìm đối tượng danh mục trong Database bằng Id
    var category = _context.Categories.Find(id);

    // Kiểm tra nếu tìm thấy thì mới xóa
    if (category != null)
    {
        // Bước 2: Lệnh xóa khỏi bộ nhớ tạm (Tracking)
        _context.Categories.Remove(category);

        // Bước 3: Chốt phiên làm việc, xóa thực sự trong SQL Server
        _context.SaveChanges();
    }

    // Sau khi xóa xong, quay lại trang danh sách để cập nhật giao diện
    return RedirectToAction("Index");
}

B. Giải thích các câu lệnh
●	_context.Categories.Find(id): Trước khi xóa, hệ thống phải "cầm" được đối tượng đó trong tay. Lệnh này tìm kiếm theo khóa chính (Id).
●	_context.Categories.Remove(category): Lệnh này chỉ mới đánh dấu đối tượng là "Sẽ bị xóa". Dữ liệu trong SQL Server lúc này vẫn còn nguyên.
●	_context.SaveChanges(): Đây là lệnh quan trọng nhất. EF Core sẽ tạo ra câu lệnh SQL DELETE FROM Categories WHERE Id = ... và gửi xuống máy chủ SQL để xóa thực sự.

3.4. THỰC HIỆN CHỈNH SỬA DỮ LIỆU (UPDATE)
Quy trình chỉnh sửa gồm 2 giai đoạn:
1.	Giai đoạn 1 (GET): Tìm danh mục theo ID và hiện thông tin lên Form để  sửa.
2.	Giai đoạn 2 (POST): Nhận dữ liệu đã sửa từ Form và cập nhật vào SQL Server.
A. Mã nguồn xử lý tại CategoryController.cs
hướng dẫn  thêm 2 hàm sau vào Controller:
C#
// 1. Hàm GET: Tìm dữ liệu cũ và đổ lên Form
[HttpGet]
public IActionResult Edit(int id)
{
    // Tìm danh mục trong Database theo Id [cite: 348, 350]
    var category = _context.Categories.Find(id);

    if (category == null) return NotFound();

    return View(category); // Gửi đối tượng tìm được sang giao diện Edit
}

// 2. Hàm POST: Nhận dữ liệu mới từ người dùng và lưu lại
[HttpPost]
public IActionResult Edit(Category model)
{
    // Lệnh cập nhật đối tượng vào bộ nhớ tạm
    _context.Categories.Update(model);

    // Lưu thay đổi thực sự xuống SQL Server [cite: 504, 509]
    _context.SaveChanges();

    // Quay lại trang danh sách để xem kết quả
    return RedirectToAction("Index");
}

B. Giải thích các lệnh chính 
●	_context.Categories.Find(id): Dùng để lấy đúng dòng dữ liệu cần sửa từ Database lên bộ nhớ.
●	_context.Categories.Update(model): Entity Framework sẽ tự động so sánh dữ liệu cũ và mới để chuẩn bị câu lệnh SQL UPDATE.
●	return View(category): Giúp các ô nhập liệu trên trang Sửa tự động hiển thị tên và mô tả hiện tại của danh mục đó.

Yêu cầu 2: HIỂN THỊ 3 BẢN TIN MỚI NHẤT LÊN TRANG CHỦ
Chúng ta sẽ kết hợp các hàm LINQ đã học là OrderByDescending và Take.
 
Mục tiêu là lọc từ cơ sở dữ liệu ra 3 bài viết có ngày đăng (CreatedDate) gần đây nhất để trình bày lên trang chủ.
Bước 1: Chỉnh sửa Controller (Controllers/HomeController.cs)
Mở HomeController.cs, tiêm ApplicationDbContext vào và viết logic lọc dữ liệu:
C#
using Microsoft.AspNetCore.Mvc;
using Microsoft.EntityFrameworkCore;
using CMS.Data; // Thư mục chứa DbContext [cite: 568]
using System.Linq;

public class HomeController : Controller
{
    private readonly ApplicationDbContext _context;

    public HomeController(ApplicationDbContext context)
    {
        _context = context;
    }

    public IActionResult Index()
    {
        // LINQ: Lấy 3 bài viết mới nhất
        var latestPosts = _context.Posts
                          [cite_start].Include(p => p.Category) // Lấy kèm tên danh mục để hiển thị 
                          [cite_start].OrderByDescending(p => p.CreatedDate) // Sắp xếp ngày mới nhất lên đầu 
                          .Take(3) // Chỉ lấy đúng 3 bản tin đầu tiên
                          .ToList();

        return View(latestPosts);
    }
}

Bước 2: Giải thích cú pháp LINQ 
●	.OrderByDescending(p => p.CreatedDate): Sắp xếp toàn bộ bài viết theo thứ tự thời gian giảm dần (mới nhất đứng trước).
●	.Take(3): Đây là lệnh "cắt" lấy đúng 3 dòng đầu tiên của danh sách đã sắp xếp.
●	.Include(p => p.Category): Đảm bảo lấy được tên danh mục (ví dụ: "Tin Công Nghệ") để hiển thị trên thẻ bài viết.
